✅ Verify Installation

💡 3 Ways to Verify

Let's make sure everything is working correctly. Try all three methods!

Method 1: Check Windows Service

Windows Services - Verify MongoDB is Running
1. Press Windows + R
2. Type: services.msc
3. Press Enter
4. Look for "MongoDB" in the list
5. Status should be "Running"
🖼️
Windows Services - MongoDB Running

Method 2: Command Line Test

CMD / PowerShell - Check MongoDB Version
# Open Command Prompt or PowerShell
# Type this command:

mongod --version

# You should see output like:
# db version v8.0.x
# Build Info: ...
# OpenSSL version: ...
⚠️ If Command Not Found

If you get "mongod is not recognized", you need to add MongoDB to your PATH. See troubleshooting section below.

Method 3: MongoDB Compass

Open MongoDB Compass:

  1. Search for "MongoDB Compass" in Windows Start Menu
  2. Click to open
  3. You should see connection screen with mongodb://localhost:27017
  4. Click "Connect"
  5. If connection succeeds, MongoDB is running! 🎉
🖼️
MongoDB Compass - Connection Screen

🚀 Your First MongoDB Commands

🎉 Congratulations!

MongoDB is installed and running. Let's execute your first database commands!

Step 1: Open MongoDB Shell (mongosh)

Open CMD/PowerShell and run mongosh
# In Command Prompt or PowerShell:
mongosh

# You'll see something like:
# Current Mongosh Log ID: ...
# Connecting to: mongodb://127.0.0.1:27017
# Connected to MongoDB version 8.0.x
# test>

Step 2: Create Your First Database

Create "myFirstDB" database
# Switch to new database (creates it automatically)
use myFirstDB

# Output: switched to db myFirstDB

Step 3: Insert Your First Document

Add a document to "students" collection
// Insert your first student
db.students.insertOne({
  name: "Rahul Kumar",
  age: 20,
  course: "Computer Science",
  marks: 85
})

// Output:
// {
//   acknowledged: true,
//   insertedId: ObjectId("...")
// }

Step 4: Query Your Data

Find all students
// Find all documents in students collection
db.students.find()

// Output: Shows the document you just inserted!
// {
//   _id: ObjectId("..."),
//   name: "Rahul Kumar",
//   age: 20,
//   course: "Computer Science",
//   marks: 85
// }

Step 5: Add More Documents

Insert multiple students at once
// Insert many documents
db.students.insertMany([
  {
    name: "Priya Sharma",
    age: 21,
    course: "Data Science",
    marks: 92
  },
  {
    name: "Amit Patel",
    age: 22,
    course: "Computer Science",
    marks: 78
  },
  {
    name: "Sneha Reddy",
    age: 20,
    course: "AI & ML",
    marks: 88
  }
])

// Output:
// {
//   acknowledged: true,
//   insertedIds: {
//     '0': ObjectId("..."),
//     '1': ObjectId("..."),
//     '2': ObjectId("...")
//   }
// }

Step 6: Try Some Queries

Filter and sort your data
// Find students with marks > 80
db.students.find({ marks: { $gt: 80 } })

// Find and sort by marks (descending)
db.students.find().sort({ marks: -1 })

// Count total students
db.students.countDocuments()

// Find one specific student
db.students.findOne({ name: "Priya Sharma" })
🎉 Amazing Work!

You just created a database, inserted documents, and ran queries. You're officially a MongoDB user now! 🚀

🔧 Troubleshooting Common Issues

⚠️ Stuck? Don't Worry!

Here are solutions to the most common Windows installation problems.

❌
'mongod' is not recognized as an internal or external command
▼

Problem: MongoDB is not in your system PATH.

Solution - Add MongoDB to PATH:

  1. Press Windows + R
  2. Type: sysdm.cpl and press Enter
  3. Click "Advanced" tab
  4. Click "Environment Variables"
  5. Under "System variables", find and select "Path"
  6. Click "Edit"
  7. Click "New"
  8. Add: C:\Program Files\MongoDB\Server\8.0\bin
  9. Click OK on all windows
  10. Close and reopen Command Prompt/PowerShell
Verify PATH is set
# Close and reopen CMD, then run:
mongod --version

# Should now work!
🚫
MongoDB service won't start / Failed to start
▼

Common Causes:

  • Port 27017 is already in use
  • Data directory doesn't exist or no permissions
  • Another MongoDB instance is running

Solution 1 - Check Port:

Check what's using port 27017
# In Command Prompt (as Administrator):
netstat -ano | findstr :27017

# If something is using port, kill it:
taskkill /PID [process_id] /F

Solution 2 - Recreate Data Directory:

Create data folder with permissions
# In PowerShell (as Administrator):
mkdir "C:\Program Files\MongoDB\Server\8.0\data"
mkdir "C:\Program Files\MongoDB\Server\8.0\log"

# Then restart MongoDB service:
net stop MongoDB
net start MongoDB
🔒
Access Denied / Permission Errors
▼

Problem: You don't have Administrator privileges.

Solution:

  1. Right-click on Command Prompt or PowerShell
  2. Select "Run as Administrator"
  3. If still issues, disable antivirus temporarily
  4. Make sure Windows Defender isn't blocking MongoDB
⚠️ Antivirus Interference

Some antivirus programs block MongoDB. Add MongoDB folder to exceptions: C:\Program Files\MongoDB\

💾
MongoDB Compass won't connect
▼

Problem: Can't connect to localhost:27017

Solutions:

1. Verify MongoDB Service is Running:

Check MongoDB status
# PowerShell:
Get-Service MongoDB

# Status should be "Running"
# If stopped, start it:
Start-Service MongoDB

2. Check Connection String:

Make sure you're using correct connection string:

mongodb://localhost:27017

OR

mongodb://127.0.0.1:27017

3. Firewall Issues:

Add MongoDB to Windows Firewall exceptions if blocked.

📦
Installation hangs at "Installing MongoDB Compass"
▼

Problem: Installer stuck downloading Compass.

Solutions:

  1. Be Patient: It can take 10-15 minutes on slow connections
  2. Check Internet: Installer downloads Compass from internet
  3. Alternative: Cancel installer, reinstall WITHOUT Compass checkbox
  4. Manual Install: Download Compass separately from MongoDB Compass Download Page
