📐 Database Schema Design

Our chat application consists of 6 interconnected collections:

👥 users - User Profiles

Store user information, authentication, and presence data

  • _id ObjectId
  • username String (unique, indexed)
  • email String (unique, indexed)
  • displayName String
  • avatar String (URL)
  • status String (online/away/busy/offline)
  • lastSeen Date (indexed)
  • bio String
  • passwordHash String
  • createdAt Date
  • isActive Boolean

💬 conversations - Chat Conversations

Manage both direct messages and group chats

  • _id ObjectId
  • type String (direct/group) (indexed)
  • name String (for groups)
  • description String (for groups)
  • avatar String (for groups)
  • participants Array [ObjectId] (indexed)
  • createdBy ObjectId (reference)
  • admins Array [ObjectId] (for groups)
  • lastMessage Object {text, senderId, timestamp}
  • createdAt Date
  • isActive Boolean

✉️ messages - Chat Messages

Store all messages with delivery tracking

  • _id ObjectId
  • conversationId ObjectId (reference, indexed)
  • senderId ObjectId (reference, indexed)
  • content String (text indexed)
  • messageType String (text/image/file/video)
  • attachments Array [{url, filename, size, type}]
  • replyTo ObjectId (reference to message)
  • reactions Array [{userId, emoji}]
  • readBy Array [{userId, readAt}]
  • deliveredTo Array [{userId, deliveredAt}]
  • isEdited Boolean
  • editedAt Date
  • isDeleted Boolean
  • createdAt Date (indexed)

📁 files - Shared Files & Media

Track file uploads and metadata

  • _id ObjectId
  • filename String
  • originalName String
  • mimeType String
  • size Number (bytes)
  • url String
  • uploadedBy ObjectId (reference)
  • conversationId ObjectId (reference, indexed)
  • messageId ObjectId (reference)
  • uploadedAt Date

⌨️ typingIndicators - Real-Time Typing Status

Track who is currently typing in conversations

  • _id ObjectId
  • conversationId ObjectId (reference, indexed)
  • userId ObjectId (reference)
  • isTyping Boolean
  • lastUpdated Date (TTL indexed)

🔔 notifications - User Notifications

Manage unread message notifications

  • _id ObjectId
  • userId ObjectId (reference, indexed)
  • conversationId ObjectId (reference)
  • messageId ObjectId (reference)
  • type String (message/mention/invite)
  • isRead Boolean (indexed)
  • createdAt Date
🎯 Key Design Decisions:
  • Conversation-centric: Both DMs and groups use same conversation collection
  • Embedded receipts: Read/delivered status embedded in messages for atomic updates
  • Text indexing: Full-text search on message content for powerful search
  • TTL for typing: Auto-cleanup of stale typing indicators using TTL index
  • Separate files: File metadata tracked separately for analytics and quotas

🔧 Initial Setup

1Create the Database

Initialize ChatFlow Database 
use chatFlow

// Verify creation
db.getName()
// Output: "chatFlow"

2Create Collections

db.createCollection("users")
db.createCollection("conversations")
db.createCollection("messages")
db.createCollection("files")
db.createCollection("typingIndicators")
db.createCollection("notifications")

3Create Text Search Indexes

Critical: Full-text search for messages 
// Message content search (most important!)
db.messages.createIndex({ content: "text" })

// User search by name and username
db.users.createIndex({ 
  username: "text", 
  displayName: "text" 
})

4Create Performance Indexes

// Users
db.users.createIndex({ username: 1 }, { unique: true })
db.users.createIndex({ email: 1 }, { unique: true })
db.users.createIndex({ lastSeen: -1 })
db.users.createIndex({ status: 1 })

// Conversations
db.conversations.createIndex({ participants: 1 })
db.conversations.createIndex({ type: 1 })
db.conversations.createIndex({ "lastMessage.timestamp": -1 })

// Messages (compound index for efficient queries)
db.messages.createIndex({ conversationId: 1, createdAt: -1 })
db.messages.createIndex({ senderId: 1, createdAt: -1 })
db.messages.createIndex({ conversationId: 1, isDeleted: 1, createdAt: -1 })

// Files
db.files.createIndex({ conversationId: 1 })
db.files.createIndex({ uploadedBy: 1 })
db.files.createIndex({ uploadedAt: -1 })

// Typing indicators (TTL index - auto-delete after 10 seconds)
db.typingIndicators.createIndex({ conversationId: 1 })
db.typingIndicators.createIndex({ lastUpdated: 1 }, { expireAfterSeconds: 10 })

// Notifications
db.notifications.createIndex({ userId: 1, isRead: 1 })
db.notifications.createIndex({ userId: 1, createdAt: -1 })
✅ Setup Complete!

Your ChatFlow database is now ready with all collections, text search indexes, and performance optimizations configured!

✨ Phase 4: Advanced Message Features

Implement typing indicators, message search, and threading.

Typing Indicators

Track Who's Typing 
// User starts typing
db.typingIndicators.updateOne(
  { 
    conversationId: conversation._id,
    userId: alice._id
  },
  {
    $set: {
      isTyping: true,
      lastUpdated: new Date()
    }
  },
  { upsert: true }
)

// User stops typing
db.typingIndicators.updateOne(
  { 
    conversationId: conversation._id,
    userId: alice._id
  },
  {
    $set: {
      isTyping: false,
      lastUpdated: new Date()
    }
  }
)

// Get who's currently typing (excluding current user)
db.typingIndicators.aggregate([
  {
    $match: {
      conversationId: conversation._id,
      userId: { $ne: alice._id },
      isTyping: true
    }
  },
  {
    $lookup: {
      from: "users",
      localField: "userId",
      foreignField: "_id",
      as: "user"
    }
  },
  { $unwind: "$user" },
  {
    $project: {
      _id: 0,
      username: "$user.username",
      displayName: "$user.displayName"
    }
  }
])

Search Messages

Full-Text Message Search 
// Search all messages for keywords
db.messages.find({
  $text: { $search: "design mockups" },
  isDeleted: false
})
.sort({ createdAt: -1 })
.limit(20)

// Search within specific conversation
db.messages.find({
  conversationId: groupChat._id,
  $text: { $search: "meeting" },
  isDeleted: false
})

// Search with relevance score
db.messages.find(
  {
    $text: { $search: "project alpha" },
    isDeleted: false
  },
  {
    score: { $meta: "textScore" }
  }
)
.sort({ score: { $meta: "textScore" } })
.limit(10)

Get Unread Message Count

// Count unread messages for a user in a conversation
db.messages.countDocuments({
  conversationId: conversation._id,
  senderId: { $ne: alice._id },  // Not sent by user
  "readBy.userId": { $ne: alice._id },  // Not read by user
  isDeleted: false
})

// Get unread count per conversation
db.messages.aggregate([
  {
    $match: {
      senderId: { $ne: alice._id },
      "readBy.userId": { $ne: alice._id },
      isDeleted: false
    }
  },
  {

Response Time Analysis

// Average time between messages in conversations
db.messages.aggregate([
  { $match: { conversationId: groupChat._id, isDeleted: false } },
  { $sort: { createdAt: 1 } },
  {
    $setWindowFields: {
      partitionBy: "$conversationId",
      sortBy: { createdAt: 1 },
      output: {
        prevTimestamp: {
          $shift: {
            output: "$createdAt",
            by: -1
          }
        }
      }
    }
  },
  {
    $match: { prevTimestamp: { $ne: null } }
  },
  {
    $project: {
      timeDiff: {
        $divide: [

          60000  // Convert to minutes
        ]
      }
    }
  },
  {
    $group: {
      _id: null,
      avgResponseTime: { $avg: "$timeDiff" },
      medianResponseTime: { $median: { input: "$timeDiff", method: "approximate" } }
    }
  },
  {
    $project: {
      _id: 0,
      avgResponseMinutes: { $round: ["$avgResponseTime", 1] },
      medianResponseMinutes: { $round: ["$medianResponseTime", 1] }
    }
  }
])

⚠️ Performance Best Practices:
  • Always use compound indexes for frequently queried field combinations
  • Implement cursor-based pagination instead of skip/limit for large datasets
  • Use projections to limit returned fields in list views
  • Consider read replicas for analytics queries on high-traffic systems
  • Archive old messages to a separate collection after 1-2 years
  • Cache conversation lists in Redis for active users
